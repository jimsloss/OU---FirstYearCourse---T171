Exercise 25: A first web page

This was the first step in learning how to prepare HTML documents, taking an existing page, modifying it a little bit and then put it on the web.  By doing this we learnt to use the basic tools needed to create more complex HTML documents.

For simplicity, I used WordPad, as suggested by the tutor, (found from Start - Programs - Accessories - then WordPad).

For help it was suggested to go to the T171 Web HTML conference.
 

Making changes to an HTML file

We began by looking at a document template in the browser.

The first step was to look at the HTML that described the template.  To do this, with the template open in the browser, I selected View source, in my browser.

I noticed some HTML tags, of the form '<sometext>', mixed in with the content of the page, it was fairly easy to distinguish the text from the tags.

The copy of the template I was viewing was stored in Milton Keynes on an Open University Web server.  I could view its HTML source through the browser, but I could not make changes to it.

To enable me to make changes, I had to save a copy of the template onto my hard disk, giving my copy the filename "templ1.htm".  (The suffixes '.htm' or '.html' are normally used for HTML files.)  To do this, having the template open in my browser I selected Save as from the File menu, and typed in the name "templ1.htm", noting where it was saved.

In my browser I then opened the file, (as opposed to opening a web location), via File - Open, which gave a file dialogue box.  Now I was looking at my own local copy of the file, which I was then going to change, rather than at the one on the OU Web server, which I couldn’t change.

I then opened WordPad.  In it I opened the copy of Template 1 I had just saved on my hard disk, ensuring that I changed the drop-down File-type box to ‘all files’.

I was able to see the same HTML source as I had seen in my browser.

I then made some minor changes, typing my name and my tutor's name where indicated, before saving the file again, making sure that the Save As Type box said Text document.  (If you receive a warning that doing this will lose all formatting, don't worry.)

Keeping WordPad open, I switched back to my browser, and to see the changes I had just made, selected 'refresh'.

My browser now displayed Template 1 with my name and my tutor's name inserted.  (It’s a good idea to keep checking your browser to ensure things are working out as expected.
 

Publishing the file on the web

The final step in this exercise was to publish the page I had just changed on the Web.
You can do this in two ways:

Through your ISP, if your contract includes web space.
Through FirstClass

I used FirstClass (as was recommended) to publish my page on the web.  To do this I had to access the OU server through the FC client instead of through your browser.

If your ISP provided you with web space, you could use that, they should also have provided instructions on how to put pages onto their web servers, and on-line or telephone help.  Be sure you know the URL of the page after you have put it up.

Using the Home Page area in FirstClass I had to do the following:

Connect to the OU server through the FC client, choose File - Open - Home Page.

A Home Page Folder icon appeared on my Desktop, you can only have one.  You can rename or delete it, but you should not move it off the Desktop, it must stay on to work properly.

I opened the Home Page Folder, and uploaded the file I had been working on, via File - Upload - and then selecting the file in the dialogue box which appeared.

I then disconnected from the FC server.
 

Finally, I had to check that it was live on the web using my browser.
 

I opened my browser and viewed the file I had just uploaded, being on the OU FirstClass server, the URL was:

http://oufcnt5.open.ac.uk/~jim_sloss/temp1.htm
